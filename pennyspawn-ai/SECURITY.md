# Security policy

## Never commit

- wallet seed phrases;
- private keys;
- Cloudflare API tokens;
- the `ADMIN_TOKEN` value;
- passwords or recovery codes.

The Worker only needs a **public receiving wallet address**.

## Safety properties

- No withdrawal or trading code.
- No borrowing or lending code.
- No automatic account creation.
- No automatic deployment of clones.
- No guaranteed-income wording.
- Obvious scam, phishing, credential-theft, malware, impersonation, fake-review, counterfeit, stolen-goods, evasion, and wallet-secret requests are blocked.

Keyword filtering is an additional barrier, not a perfect moderation system. Keep logs minimal and review abuse reports.

## Reporting

Open a GitHub issue without including secrets, exploit payloads, personal information, or wallet credentials.
